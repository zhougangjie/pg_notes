## 数据库的数据面和控制面

将数据库系统彻底规整为**数据面（Data Plane）** 与 **控制面（Control Plane）** 两大阵营：

**1. 数据面 (Data Plane)：物理实体的流转**

数据面是数据库的“肉体”，负责处理所有与字节、块、文件相关的物理操作。它的核心目标是：极速读写、持久存储。

- 存储引擎（Storage Engine）：管理数据在磁盘上的物理布局（Heap, B-Tree）。
- 缓冲池（Shared Buffers）：数据在内存中的镜像，是数据流转的中转站。
- 物理辅助（FSM/VM）：空闲空间映射（FSM）和可见性映射（VM），它们是数据页的“物理索引”。
- 日志实体（WAL Segments）：数据变化的物理流水账。

**2. 控制面 (Control Plane)：逻辑秩序的守护**

控制面是数据库的“灵魂”，负责处理所有与规则、状态、决策、同步相关的逻辑操作。

A. 决策与规则 (Decision & Schema)

- 查询解析与优化（Optimizer）：基于元数据（Syscache）和统计信息，制定数据流转的最优路径。
- 安全与权限：控制谁能接触数据面。

B. 状态与并发 (State & Concurrency)

- 事务管理（XID/CLOG）：维护事务的生命周期，标记数据的逻辑版本。
- 多版本控制（MVCC）：逻辑上的“时空分割”，确保不同事务看到不同的数据快照。

C. 运行支撑与底层同步（Runtime Infrastructure）

- 锁管理器（Lock Manager）：重型锁确保业务逻辑不冲突（如：禁止在插入时删表）。
- 同步原语（LWLocks/Spinlocks）：轻量级锁确保控制面在修改共享状态（如修改事务号、分配内存）时的物理原子性。
- 内存上下文（MemoryContext）：确保控制面在运行过程中，内存资源的分配和回收是合理的、不泄露的。

| 维度     | 控制面 (Control Plane)                            | 数据面 (Data Plane)                    |
| -------- | ------------------------------------------------- | -------------------------------------- |
| 关注点   | 逻辑（Logic）：能不能做？谁在做？                 | 物理（Physical）：存哪里？怎么搬？     |
| 核心职责 | 制定计划、管理冲突、维护状态、分配资源。          | 执行读写、管理缓存、保证落盘。         |
| 典型模块 | Optimizer, Lock Manager, Syscache, Runtime, MVCC. | Buffer Manager, Heap/Index, WAL, Disk. |
| 冲突处理 | 锁等待、死锁检测、事务回滚。                      | 页面置换、I/O 队列等待。               |

归根结底的逻辑：

- 数据面是受控者，它只管高效地读写 Page。
- 控制面是管理者，它通过 Runtime（内存/锁/进程）提供的手段，确保数据面的操作符合 ACID 规则。

**所有的“锁”和“事务状态”，本质上都是控制面为了防止数据面陷入“无序熵增”而加的人为约束。**

## 锁

### 1. **锁在底层到底是什么？**

- 在内存实现层面，锁本质上是共享内存中的一堆 Struct（结构体）。
- 它并不存在于数据文件中，也没有对应的 Page。它只存在于**共享内存（Shared Memory）**中一个专门划分出来的区域——锁表（Lock Table）。
- 它的本质：是一个记录了“谁（进程）正在对什么（对象）做什么（模式）”的多维哈希表。
- 它的表现：当一个进程想操作某个对象时，它必须去这张哈希表里“注册”一下。如果表里已经有人注册了不兼容的行为，你就必须原地等待。

### 1. **锁是如何代表“逻辑”的？**

**A：锁对象标识 (LOCKTAG)**

锁如何代表一张特定的表？

通过一个名为 LOCKTAG 的唯一结构。它就像是数据库对象的“身份证号”。它包含：Database OID + Relation OID + LockTagType（标识是表、还是页、还是事务）。

逻辑含义：只要两个进程生成的 LOCKTAG 字节序列完全一样，它们在锁表中就对应同一个“逻辑槽位”。

**B：冲突矩阵 (Conflict Matrix)**

锁如何代表“不能同时操作”的逻辑？

通过一个 位图（Bitmap）。内核中定义了一个二维数组（矩阵），横轴是“已持有模式”，纵轴是“申请模式”。

逻辑含义：如果 Matrix[已持有][申请] 为真，逻辑上就判定为“冲突”。这种逻辑不是算出来的，是程序员根据数据库理论硬编码进去的规则。

**C：等待队列 (Wait Queue)**

锁如何代表“顺序”？

通过一个 双向链表。如果检测到冲突，你的 PGPROC（进程控制块）会被挂在这个锁对象的等待队列尾部。

逻辑含义：你的进程会进入休眠状态，直到排在你前面的人修改了锁状态并发送“信号量（Semaphore）”把你唤醒。
