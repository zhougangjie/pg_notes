# Overview

## 执行模型

> **PL/pgSQL = 解释执行(晚绑定) + SQL 内核执行(SPI)**

## 执行流程

```text
调用函数
→ 首次编译（解析成语句树 + 保存 SQL 字符串）
→ 解释器逐条执行语句
→ 遇到 SQL：SPI 调用 SQL 引擎执行（缓存静态 SQL 执行计划）
→ 返回结果
```

## 关键点

- 创建: 存储过程时仅检查语法正确性，无语义分析，不建立硬依赖( pg_depends 无条目)
- 执行: (PL/pgSQL解释器)解析并执行，遇到 SQL 通过 SPI 调用 SQL 引擎执行(缓存静态 SQL 执行计划)
- 依赖: 运行时绑定; 软依赖
	- 静态 SQL: 运行时绑定;
	- 动态 SQL: 每次运行绑定;
	- 函数调用: 运行时绑定
	- 变量类型: 运行时绑定

## 虚谷

- 创建: 编译: 语法分析、语义分析、注册硬依赖
- 执行: 首次执行时编译
- 依赖: **编译时**绑定; 硬依赖(dba_depends)
